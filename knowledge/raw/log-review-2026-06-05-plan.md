### #2 ‚ÄĒ Lot card must not fail on photo (text-only fallback)
On photo upload / attachment failure (incl. VK 100), retry `video.createComment`
**without** attachments so the card always publishes (price + code + placeholder
line). Collect a list of articles with broken/missing photos for the operator.
Files: `server/vk.js` (`publishLotCard` ~424-458, `buildLotCardMessage` ~99).
Operator action: add photos to 00136, 00037, 03018, 03232.

### #1 ‚ÄĒ Adaptive comment poll + publish priority
Root cause is contention on one VK token, not lot count (polling is a single
global `getComments`, not per-lot). Two changes:
- Dynamic poll interval (‚Čą1.5 s when comments are flowing, 5‚Äď8 s when quiet)
  instead of fixed 2 s ‚ÄĒ `pollLoop` `server/ws-server.js` ~1330/1604.
- Two-lane VK queue: publishing (card/price/reservation reply/close) takes
  priority over polling ‚ÄĒ `enqueueVkApiCall` `server/vk.js` ~208-247.
Higher-ceiling alternative for later: VK Bots Long Poll (`video_comment_new`)
to remove read traffic entirely.
